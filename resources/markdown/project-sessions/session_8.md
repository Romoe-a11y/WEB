**Session 8  —  Testing, Debugging, Final Improvements, and Demo Preparation** 

**Objective** 

Finalize the project, fix remaining issues, polish the interface, and prepare a confident, wellstructured demonstration. 

## **1.  Why Testing Matters** 

A project is never truly finished until it has been tested carefully. Testing means trying every feature, edge case, and potential failure point. This week you verify that all parts of the application work correctly together. 

## **2.  Testing Checklist** 

||||
|---|---|---|
|**#**|**Feature to Test**|**What to Verify**|
|**1**|**Registration**|New user is saved in the database|
|**2**|**Login**|Correct credentials log in; wrong ones show<br>an error|
|**3**|**Add Module**|Module appears in the list after submission|
|**4**|**Display Modules**|Only the logged-in user's modules are shown|
|**5**|**Edit Module**|Changes are saved and visible immediately|
|**6**|**Delete Module**|Confirmation is asked; module is removed|
|**7**|**AI Plan**|Plan is generated and displayed correctly|
|**8**|**Logout**|Session is destroyed; user is redirected to<br>login|



## **3.  Suggested Demo Scenario** 

## **Follow this order during your demo:** 

1. Register a new user account 

2. Log in with the new credentials 

3. Add 3-4 modules with different difficulty levels and exam dates 

4. Edit one module — change its difficulty or progress 

5. Delete one module with the confirmation step 

6. Generate the AI revision plan and show the result 

7. Log out and verify the session is closed

## **4.  Practical Tasks This Week** 

- **1** Test every item in the testing checklist above 

- **2** Fix all bugs discovered during testing 

- **3** Clean and organize your PHP files and folder structure 

- **4** Remove debug output (echo, var_dump) from your code 

- **5** Make sure localhost works correctly before presentation day 

- **6** Prepare a clear browser window with a clean demo account 

- **7** Practice the demo scenario 2-3 times out loud 

## **Expected Result by End of Week** 

> ✓ The full application works without errors 

- ✓ All 8 test items are verified and passing 

> ✓ The code is organized and the interface is clean 

- ✓ You are ready to present confidently 

## **Advice & Common Mistakes to Avoid** 

- **!** Demonstrate the CRUD first — show the AI after, not before 

- **!** Use a real, well-filled demo account with meaningful module data 

- **!** Do not try to add new features at the last minute — focus on stability 

- **!** Make sure your laptop charger is available on presentation day
