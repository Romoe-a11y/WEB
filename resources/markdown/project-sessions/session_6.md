## **Session 6  —  Dashboard, Styling, and User Experience** 

**Objective** 

Improve the application's visual design and usability. Build a meaningful dashboard that gives the student a useful overview of their modules. 

## **1.  Why Good Interface Design Matters** 

A functional application needs to be readable and easy to use. This week you focus on HTML5 structure and CSS3 styling to make the application feel polished and professional. Good design does not mean complex — it means clear, consistent, and readable. 

## **2.  Dashboard Ideas** 

Your dashboard.php should give the student useful information at a glance: 

- Total number of modules registered 

- The module with the nearest exam date 

- The module with the highest difficulty 

- Average progress across all modules 

- A direct button to Add a new module or Generate the AI plan

## **Advice & Common Mistakes to Avoid** 

**!** Do not overcomplicate the design — clarity is more important than decoration 

**!** Use an external CSS file (style.css) and link it on every page with <link> 

**!** Avoid inline styles — keep CSS in one place for easy maintenance 

**!** Simple HTML cards with colored borders look professional and are easy to build 

**!** Good design is simple design
